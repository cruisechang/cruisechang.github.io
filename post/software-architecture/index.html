software architecture

這篇紀錄軟體架構的的各種觀念。

軟體架構師的責任

  • 理解架構需求與限制

    理解業務目標，管理架構驅動力
滿足「功能需求」及「非功能需求」及「環境限制」

  • 設計軟體

    依據業務、需求建立軟體系統的的整體架構

  • 技術選擇(技術風險管理)

    選擇「合用的技術」
技術的討論、驗證、測試
未來不會有問題的技術

  • 架構演進

    架構的持續更迭

  • 編寫程式

    持續寫程式，實踐自己做的架構

  • 保證品質

    進行要求，及使用各種工具、技術，確保軟件品質
例如會議、unittest、code review、swagger、flow 規範
訂定各種標準
確保團隊對架構一致執行

軟體架構師的能力

  • 他是通才型專家

    須知道多種技術
深度與廣度並存
著眼大局觀，而非只看小處
他不一定是團隊中程式寫得最好的人
對於不知道的事，找到能合作的人才是方法
他能回答類似問題：為什麼這個技術合適？這個決策的利弊？系統設計有哪些選擇？
如何證明這個架構有效？如何確保品質？

  • 具備軟技能

    領導力：創造願景，帶領團隊像共同目標前進
溝通：有效像團隊溝通你的設計、想法、願景
影響力：用勸說、談判、操控去讓團隊追求你要的結果
信心：是一切基礎
合作：與他人合作
指導：對不了解的人指導
輔導：對某人進行學習指引
動力：保持團隊愉快、開朗、積極。要面對團隊中一些人不買賬
潤滑劑：不同意見時要想法達成共識
政治：處理政治
責任感：不逃避責任
授權：授權給其他人

  • 有時軟技能比技術重要

    和諧的團隊才能做事，團隊內鬨一切都免談
